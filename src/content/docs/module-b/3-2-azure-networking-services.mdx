## What the Auditor Checks

The auditor verifies you have implemented **at least five (5) native Azure networking services** from the list below across one or more customer connectivity projects within the last **18 months**.

### Eligible services (need ≥ 5)

- Virtual Network and VNet Peering
- ExpressRoute
- VPN
- Virtual WAN
- CDN
- Front Door
- Load Balancer
- Private Link
- Traffic Manager
- Application Gateway
- Azure Firewall

**Typical questions:**
- Which five services have you implemented in the last 18 months?
- Show me the design and as-built / SOW for each.
- Which customer used which service?

## Evidence Guidance

### Service-to-customer matrix
A single sheet mapping each implemented service to: customer (anonymised), engagement date, and the document(s) where it appears.

### Document evidence
Same accepted document types as B.3.1. Architecture diagrams should label each service explicitly so the auditor can count them at a glance.

## Common Gaps

- **Counting closely-related services as one:** ExpressRoute and VPN are separate services; same with App Gateway and Front Door. Count them separately.
- **Diagrams without service labels:** Auditor can't verify what they can't see — label every component with its Azure service name.
- **Older engagements:** &gt; 18 months disqualifies — refresh with a current engagement.
