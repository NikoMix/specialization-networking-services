## When to use this
Use during the design phase to anchor the customer's solution to a known-good starting point. Reference Architecture choice is captured in the [Discovery Workshop decision log](../discovery-workshop/).

## How each reference page is structured

| Section | Purpose |
|---|---|
| **Context** | When this architecture fits the customer's profile |
| **Diagram** | Topology and traffic flow |
| **Component table** | SKU / tier rationale per component |
| **Data flow narrative** | What traverses each link, in what direction, why |
| **IaC starter link** | Bicep / Terraform / ALZ Accelerator path |
| **Known limitations** | Where this design starts to creak |
| **When to choose / when NOT to choose** | Decision criteria |

## Reuse & contribute back

New architecture? Raise a **Template Improvement** issue labelled `reference-architecture`, then PR a new MDX file under `reference-architectures/` following the structure above.
