## What is the Networking Services in Microsoft Azure Advanced Specialization?

The **Networking Services in Microsoft Azure Advanced Specialization** is a Microsoft Partner credential that validates deep technical expertise in planning, designing, and deploying Azure networking solutions — at enterprise scale, across hybrid and multi-region topologies, with strong security and observability posture.

It signals to customers and Microsoft that your organisation:

- Has **proven delivery capability** for Azure networking — hub-spoke, Virtual WAN, ExpressRoute, secure edge, multi-region traffic management
- Implements **at least five (5) native Azure networking services** across real production customer engagements
- Has passed a rigorous **independent third-party audit** (ISSI) validating both your repeatable cloud-foundation practice (Module A) and your networking workload practice (Module B)

<Aside type="tip">
  Partners holding this specialization typically appear higher in Microsoft AppSource Partner Finder, qualify for additional co-sell motions on connectivity-led deals (VWAN / ExpressRoute / Azure Firewall), and gain priority access to Microsoft-funded marketing for landing-zone connectivity offers.
</Aside>

## Audit version in scope

This guide is aligned to:

- **V1.8.1** — active **Jan 1 – Jun 30, 2026**
- **V1.9 PREVIEW** — available for partner review

A Pass result satisfies the audit requirement for **two (2) years**.

## How this guide helps

Each page maps to a specific audit control and contains:

1. **What the auditor checks** — exact criteria from the V1.8.1 checklist
2. **Required evidence** — a structured checklist of accepted document types
3. **Evidence status table** — assign owners, set deadlines, track status
4. **Tips** — practical advice specific to networking engagements
5. **Common gaps** — what tends to fail in this workload and how to avoid it

In addition, the **Engagement Playbook** packages the consultant-side artifacts (qualification questionnaire, discovery workshop, WAF assessment, reference architectures, HLD/LLD/runbook/KT/hypercare templates) so each customer engagement starts from a curated baseline rather than a blank page.

## Specialization scope

| Module | Scope |
|---|---|
| **Module A** | Azure Essentials Cloud Foundation — generic adoption methodology (Readiness, Design & Govern, Manage & Optimize). Reusable across multiple Azure specializations. |
| **Module B** | Networking Services in Microsoft Azure — Assess, Design, Deployment, Review & Release for Operations. |

## Audit administration

- **Auditor:** Information Security Systems International, LLC (**ISSI**) — independent third party.
- **Duration:** Module B = 4 hours; Module A + B combined = 8 hours.
- **Pricing:** Module B = $2,400 USD; Module A+B = $3,600 USD (Gap Review Meeting included).
- **Solutions Partner designation required:** Solutions Partner for **Infrastructure (Azure)**.
