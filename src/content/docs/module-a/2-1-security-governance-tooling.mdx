## What the Auditor Checks

The auditor verifies you can **deploy the required governance tools for compliance and security** for two (2) unique customer projects, including an established security baseline (Microsoft Defender for Cloud or 3rd party) and a completed **Cloud Adoption Security Review** assessment.

**Typical questions:**
- Show me the Defender for Cloud Secure Score baseline for each customer.
- Show me the completed Cloud Adoption Security Review for each customer.
- How do you enforce the baseline going forward?

> Projects must have completed in the past **12 months**. Equivalent 3rd-party tooling is acceptable in lieu of Defender for Cloud if it provides a comparable baseline.

<Aside type="tip">
  See the [governance tools templates](https://learn.microsoft.com/azure/cloud-adoption-framework/govern/) for the security baseline templates the auditor expects to see referenced.
</Aside>

## Evidence Guidance

### Defender for Cloud baseline
Enable Defender for Cloud on the customer subscription, capture the initial Secure Score, define the baseline targets, and export the recommendations workbook.

### Cloud Adoption Security Review
Run the [Cloud Adoption Security Review](https://learn.microsoft.com/assessments/) — export the customer-specific report and your action plan.

## Common Gaps

- **Baseline captured but no follow-through:** Show the remediation actions and the score improvement over time.
- **Security Review not exported per customer:** Generic exports do not qualify — must be customer-specific.
- **Only 3rd-party tooling, no link to Azure controls:** Map the 3rd-party findings back to specific Azure-native controls (Policy, NSGs, Defender plans).
