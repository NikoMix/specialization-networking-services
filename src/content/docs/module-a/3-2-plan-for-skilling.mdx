## What the Auditor Checks

The auditor verifies you produced a **skilling plan** for the customer's technical staff covering:

1. New skills for technical roles (IT Admins, Governance, Operations, Security)
2. Knowledge-transfer resources (Microsoft Learn paths, certifications, etc.)
3. *(For AI Applications / AI Platform on Azure specializations only — not Networking)* a TAGA report

**Typical questions:**
- Walk me through how you assess current vs target skills for the customer's team.
- Show me the skilling plan document for each customer.
- Which Microsoft Learn paths and certifications are mapped to each role?

## Evidence Guidance

### Skilling plan
Customer-facing deck or planning document. Must include: current vs target skills per role, learning paths assigned, expected duration, certification targets, success measures.

### Role-to-skill mapping
A table mapping each named role (IT Admin, IT Operations, IT Security, IT Governance, Network Engineer, etc.) to required Azure skills and learning resources.

## Common Gaps

- **Plan exists but no role-to-skill table:** Without the mapping the auditor cannot validate completeness.
- **All paths point to AZ-900:** AZ-900 is foundational only; include role-specific deeper paths (AZ-700 for network engineers, AZ-104 for ops).
- **Plan dated &gt; 12 months ago:** Re-engage and refresh.
