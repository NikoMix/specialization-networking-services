## What the Auditor Checks

The auditor verifies you can deploy Azure Landing Zones repeatably for two (2) unique customers using **Bicep, Terraform, OR ARM** (any one is sufficient), addressing at minimum:

- **Identity** — adoption of identity management (Microsoft Entra ID or equivalent)
- **Networking topology** — CAF-aligned network topology; hybrid via ExpressRoute / VPN Gateway / equivalent
- **Resource organization** — tagging and naming standards

Plus a completed **Azure Landing Zone Review** assessment confirming multi-regional or multi-zone redundancy policy is configured.

**Typical questions:**
- Show me the IaC repo or pipeline that deployed the landing zone for each customer.
- Show me the ALZ Review export for each customer.
- Which approach did you use — start-small, full ALZ, alternative, or brownfield?

<Aside type="tip">
  The **ALZ Accelerator** (Bicep, Terraform, or Azure-portal experience) is explicitly accepted. You don't need to roll your own IaC if the Accelerator covers the customer's needs.
</Aside>

## Evidence Guidance

### IaC source
Point the auditor to the repo + pipeline run that deployed the customer environment. Anonymise any tenant IDs.

### ALZ Review
Run the [Azure Landing Zone Review](https://learn.microsoft.com/assessments/) — confirm the redundancy posture (multi-region or multi-zone) and export per customer.

### Approach justification
If you deviated from the standard ALZ conceptual architecture, document the customer requirement that drove the deviation.

## Common Gaps

- **"We use IaC" claim with no repo to walk through:** The repo / pipeline must be demonstrable live.
- **No ALZ Review export:** Without it, the redundancy-policy requirement is unmet.
- **Brownfield deployment with no remediation plan:** The brownfield approach is valid but you must show the **update plan** to align to best practices.
- **Networking attributes missing:** Identity + Networking + Resource org must all be addressed unless the special Analytics / DW-Migration waiver applies (it does not apply to Networking Services).
